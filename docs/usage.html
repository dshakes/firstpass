Using Firstpass

Firstpass is a drop-in, Anthropic-compatible HTTP proxy. Run it, point your agent's base_url at it, and every request is routed to the cheapest model that provably passes your gate — with a signed receipt for each decision. This page covers install, configuration, and operation. For the design rationale, read the SPEC.

Overview

Every request flows through three moves: route to the cheapest rung of a ladder, gate the real output, and escalate one rung only if the gate fails — across providers, with cross-provider failover on a 5xx. The whole decision is written to a tamper-evident, hash-chained trace.

Quickstart

See the whole loop in ~10 seconds — no API keys. The demo stands up a mock upstream and drives one real decision, escalation and all.

# clone & watch the receipt print
git clone https://github.com/dshakes/firstpass && cd firstpass
cargo run -p firstpass-proxy --example demo

Modes

Firstpass runs in one of two modes, set by FIRSTPASS_MODE:

observe (default)

Transparent pass-through. Requests are forwarded unchanged and a trace is recorded off the hot path — zero behavior change. Use this to shadow traffic and build an audit trail before enforcing anything.

enforce

Full routing: cheapest-first ladder, gate the output, escalate on failure, cross-provider failover. Requires a routing config (below).

FIRSTPASS_MODE=enforce FIRSTPASS_CONFIG=./firstpass.toml firstpass-proxy

Routing config

Routing is declarative TOML: ordered routes (first match wins), each binding a slice of traffic to a mode, a cheapest-first model ladder, and the gates that must pass. Start from firstpass.example.toml.

[[route]]
match  = { agent = "claude-code" }
mode   = "enforce"
ladder = ["anthropic/claude-haiku-4-5", "anthropic/claude-sonnet-5"]
gates  = ["schema", "cargo-test"]

# wildcard fallback — first match wins, top to bottom
[[route]]
match = {}
mode  = "observe"

Providers

anthropic and openai are built in. A [[provider]] entry adds any other, so a ladder rung — <id>/<model> — can point anywhere. Because the OpenAI Chat Completions API is the de-facto standard, one openai-dialect entry covers Groq, Together, Fireworks, DeepSeek, Mistral, xAI, Azure, or a local Ollama / vLLM server running open-source models. gemini is its own dialect; aws_sigv4 (Bedrock) and gcp_oauth (Vertex) are bespoke-auth backends whose signing/token minting is delegated to maintained crates — credentials come from the standard env vars and never touch a URL or a log.

# any OpenAI-compatible host
[[provider]]
id          = "groq"
dialect     = "openai"
base_url    = "https://api.groq.com/openai"
api_key_env = "GROQ_API_KEY"

# Google Gemini (own dialect)
[[provider]]
id          = "gemini"
dialect     = "gemini"
base_url    = "https://generativelanguage.googleapis.com"
api_key_env = "GEMINI_API_KEY"

# AWS Bedrock (Claude on Bedrock; SigV4, creds from AWS_* env)
[[provider]]
id      = "bedrock"
dialect = "anthropic"
auth    = "aws_sigv4"
region  = "us-east-1"

# Google Vertex (Claude on Vertex; OAuth, creds from GOOGLE_APPLICATION_CREDENTIALS)
[[provider]]
id      = "vertex"
dialect = "anthropic"
auth    = "gcp_oauth"
region  = "us-east5"
project = "my-gcp-project"

# open-source rung → frontier escalation on gate fail
[[route]]
match  = {}
mode   = "enforce"
ladder = ["groq/llama-3.3-70b-versatile", "anthropic/claude-sonnet-5"]
Live status. Anthropic is live-verified; OpenAI-compatible and Gemini are wire-tested against spec-accurate servers; Bedrock and Vertex are offline-tested (shape + signed headers) and live-unverified — verify against real AWS/GCP credentials before production.

Gates

A gate reads the candidate output and returns a verdict. Four kinds:

  • Inline — non-empty, json-valid, and a minimal JSON-Schema gate (type / required / properties). No process spawn.
  • Subprocess plugin — your own command (tests, linter, a judge). The candidate is passed on stdin, never argv (injection-resistant); a JSON verdict comes back on stdout. Non-zero exit or timeout → abstain.
  • Native judge — judge = { model, threshold, rubric }: a second model grades the answer against your rubric. Maker ≠ checker is enforced structurally, and the candidate is fenced as data under an anti-injection system prompt.
  • Self-consistency — consistency = { model, k, threshold }: resample the request k times and score the candidate by agreement (self-consistency, Wang et al. 2022; semantic-entropy family). A continuous confidence score, measured on the actual output, that the conformal threshold can calibrate. Honest cost: k extra calls on the sampling model.
Error budgets. Each gate carries a rolling budget. A gate whose abstain rate exceeds its threshold is auto-disabled for subsequent requests (with an alarm), so one flaky gate can't take down a route.

CLI

One binary, firstpass, wraps the whole loop — setup, operation, audit, and the science. Every subcommand shares the same environment (above); run firstpass --help for the reference.

CommandWhat it does
onboardAgentic setup — detect env, start the proxy, route your agent, verify itself. --apply writes; bare is a dry run.
offboardUndo it — strip the rc line, stop the proxy, print the unset.
upStart the proxy (serves until Ctrl-C).
doctorValidate config, provider key, and gate binaries before you route anything.
tracePrint recent audit traces as JSON lines (--limit N).
savingsSpend vs the always-top counterfactual, from your own receipts (--json).
evalsPer-gate verdict rates + escalation + serve-by-rung, from receipts.
explainexplain <trace-id> — why one routing decision went the way it did.
exportWrite the sealed receipt log as JSONL (--out F) to hand to an auditor.
verifyIndependently re-derive the receipt hash chain (--file F); exit 1 if broken.
calibrateRecalibrate the serve threshold from deferred feedback. --method ltt gives the Learn-then-Test / risk-controlling variant; --alpha --delta --min-n tune it.
opeOff-policy eval — --config candidate.toml rehearses a policy, or --start-rung N gives an IPS/SNIPS/DR estimate, before you enforce.
mcpServe MCP over stdio — an agent reads traces, savings, evals, rehearsal, route explanations, and receipt verification as tools.

Receipts & feedback

Every decision is a hash-chained JSON trace an external auditor can re-derive — which model ran, which gates passed, what it cost, and the counterfactual saving. Downstream outcomes (did the tests pass an hour later?) flow back via POST /v1/feedback:

curl localhost:8080/v1/feedback -d '{
  "trace_id": "0192f3a1-…", "gate_id": "cargo-test",
  "verdict":  "pass", "reporter": "ci"
}'
Chain-preserving. Deferred verdicts live in a separate table keyed by trace id — they are merged on read for display but never folded into the sealed, hashed record. The chain stays re-derivable.

The self-tuning loop, end to end: calibrate a serve threshold from your receipts with firstpass calibrate (--method ltt gives the Learn-then-Test / risk-controlling variant: finite-sample, distribution-free control of served-failure at your α, plus the gate's observed false-accept rate at the chosen threshold); pin it as serve_threshold; then let [escalation.adaptive] (adaptive conformal, Gibbs–Candès) track it live from /v1/feedback outcomes. The two exported gauges — firstpass_serve_threshold and firstpass_realized_served_failure — are the loop's instruments: if realized failure drifts toward your α the threshold tightens on its own, and you can watch it happen. This is the claim that matters: a served-failure bound that holds under drift, not a one-time calibration that silently decays.

Those receipts are also an evaluation dataset. firstpass calibrate turns deferred feedback into a conformal serve threshold, and firstpass ope --config candidate.toml rehearses a policy change — replaying logged traffic against a candidate ladder/threshold and reporting estimated cost and served-failure with bootstrap confidence intervals (coverage-honest: rungs never logged are counted, not guessed) — before you enforce anything.

Offboarding

Leaving is always one environment variable — an invariant of the design:

unset ANTHROPIC_BASE_URL   # your agent talks to the provider directly again
